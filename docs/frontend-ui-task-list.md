# JobClaw 前端开发任务清单

> 更新日期：2026-03-27  
> 来源：`docs/frontend-ui-review.md`  
> 用途：将前端问题审查结果拆解为可直接执行的开发任务列表，便于逐项推进与验收。

## 1. P0 基础收敛

### 1.1 设计变量层

- [ ] 建立全局设计变量层，在 `:root` 中统一定义颜色、圆角、阴影、边框、间距、字体变量。
- [ ] 替换 `public/index.html` 与 `public/css/style.css` 中零散的颜色和阴影值，避免继续扩散硬编码样式。
- [ ] 统一主色、动作色、成功色、警告色、危险色、中性背景、中性边框、中性文本的职责。

### 1.2 字体系统

- [ ] 去掉页面根部的全局 `font-mono` 使用。
- [ ] 为正文和标题切换统一无衬线字体。
- [ ] 将等宽字体限制在日志、代码块、路径和技术信息展示场景。

### 1.3 组件状态规范

- [ ] 建立按钮状态规范：默认、hover、active、disabled、focus-visible、danger。
- [ ] 建立输入控件状态规范：默认、focus-visible、error、disabled。
- [ ] 统一导航按钮、批量按钮、保存按钮、模态按钮和链接按钮的交互样式。

### 1.4 图标系统

- [ ] 清理导航、主按钮、模态和简历区中的 emoji 语义。
- [ ] 替换为统一 SVG 图标体系。
- [ ] 保证图标仅作为辅助表达，不承担唯一语义。

## 2. P0 布局与响应式

### 2.1 主页面骨架

- [ ] 移除聊天区 `calc(100vh - 180px)` 的固定高度依赖。
- [ ] 移除职位区和配置区 `h-[calc(100vh-140px)]` 的固定高度依赖。
- [ ] 改成“头部 + 自适应内容区 + 模块内部滚动”的弹性布局。

### 2.2 头部与导航响应式

- [ ] 重做头部区布局，保证 Logo、导航和连接状态在窄屏下自然换行。
- [ ] 避免导航在 `375px` 下横向溢出或按钮过窄。
- [ ] 为头部信息区建立明确的窄屏布局规则。

### 2.3 职位页响应式

- [ ] 重做职位页顶部操作栏，避免批量按钮在小屏下拥挤成一行。
- [ ] 将批量操作区改为可换行工具栏，或拆分为主操作和更多操作。
- [ ] 确保表格区在窄屏时仍然可读且可操作。

### 2.4 配置页响应式

- [ ] 让基础设置卡片与配置编辑器在小屏下按纵向堆叠。
- [ ] 保证编辑器区域在中小尺寸窗口下仍保有足够输入空间。
- [ ] 明确配置页的内部滚动边界，避免外层与内层同时滚动。

### 2.5 页面滚动治理

- [ ] 统一主页面滚动和模块内部滚动的职责。
- [ ] 消除聊天页、职位页、配置页中明显的双滚动冲突。
- [ ] 为 `375 / 768 / 1024 / 1440` 四个断点建立基本回归检查。

## 3. P0 可访问性

### 3.1 排序表头语义

- [ ] 将职位表中可点击排序的 `<th>` 改为包含 `<button>` 的结构。
- [ ] 同步维护 `aria-sort`。
- [ ] 为当前排序字段和方向提供明确视觉提示。

### 3.2 焦点体系

- [ ] 为导航按钮补齐统一 `focus-visible`。
- [ ] 为批量操作按钮补齐统一 `focus-visible`。
- [ ] 为输入框、文本域、文件上传控件补齐统一 `focus-visible`。
- [ ] 为表格中的链接和可点击元素补齐统一 `focus-visible`。

### 3.3 模态框语义和焦点管理

- [ ] 给模态框补充 `role="dialog"` 和 `aria-modal="true"`。
- [ ] 建立标题和描述的语义关联。
- [ ] 打开模态框时聚焦首个可交互元素。
- [ ] 实现 Tab 键焦点陷阱，避免焦点跳出模态。
- [ ] 关闭模态框后恢复到触发按钮。

### 3.4 状态表达

- [ ] 确保连接状态不只依赖颜色点展示。
- [ ] 确保错误、警告、成功状态不只依赖颜色。
- [ ] 为关键状态区域增加必要的文本和无障碍语义支持。

### 3.5 动效降级

- [ ] 为聊天消息动画增加 `prefers-reduced-motion` 兼容。
- [ ] 为流式加载动画增加 `prefers-reduced-motion` 兼容。
- [ ] 去掉非必要的持续型装饰动画。

## 4. P0 任务反馈与交互闭环

### 4.1 统一反馈组件

- [ ] 实现统一消息组件，替换原生 `alert()`。
- [ ] 实现统一确认组件，替换原生 `confirm()`。
- [ ] 支持 `success / error / warning / info` 四类反馈。

### 4.2 统一任务状态模型

- [ ] 定义前端统一任务状态：已提交、排队中、执行中、完成、失败。
- [ ] 明确哪些状态显示在当前页，哪些状态进入日志区。
- [ ] 统一聊天、简历、配置保存、批量操作的反馈方式。

### 4.3 聊天区状态治理

- [ ] 修复聊天在非排队模式下直接清空历史的问题。
- [ ] 统一非流式返回和流式返回的消息进入方式。
- [ ] 明确用户消息、Agent 消息、系统提示、错误提示的展示规则。

### 4.4 简历任务闭环

- [ ] 移除简历评价任务提交后的强制切页逻辑。
- [ ] 移除简历生成任务提交后的强制切页逻辑。
- [ ] 在简历页内展示任务状态变化和结果入口。
- [ ] 为失败任务提供明确原因，而不是只在日志中体现。

### 4.5 日志区职责重构

- [ ] 明确日志区只承担系统事件和执行轨迹展示。
- [ ] 主要用户反馈不再依赖日志区承载。
- [ ] 统一日志区的事件级别和展示格式。

## 5. P1 职位看板重构

### 5.1 筛选与搜索

- [ ] 增加职位状态筛选：`all / discovered / applied / failed / favorite`。
- [ ] 增加公司名关键字搜索。
- [ ] 增加职位名关键字搜索。
- [ ] 提供最近更新时间的显式排序和过滤入口。

### 5.2 行级详情

- [ ] 为职位行增加详情预览入口。
- [ ] 实现侧边抽屉或等价详情容器。
- [ ] 在详情中展示链接、状态、更新时间和后续可扩展字段。

### 5.3 批量操作区重构

- [ ] 仅在有选中项时展示完整批量操作工具栏。
- [ ] 将危险操作与普通操作分区。
- [ ] 优化批量操作在窄屏下的布局。

### 5.4 图表联动

- [ ] 让统计图例可点击。
- [ ] 让图表扇区点击后触发表格筛选。
- [ ] 让图表从“展示结果”升级为“任务入口”。

### 5.5 空状态与无结果状态

- [ ] 区分“还没有职位数据”和“当前筛选无结果”。
- [ ] 为职位页空状态提供明确下一步引导。

### 8.1 前端依赖稳定性

- [ ] 评估将关键 CDN 依赖改为本地依赖或构建产物。

### 8.2 样式组织

- [ ] 建立页面通用样式、组件样式、状态样式的拆分规则。

## 9. 推荐执行顺序

建议按下面顺序推进，而不是并行散修：

1. 设计变量层
2. 页面骨架与响应式
3. 全局焦点态与模态可访问性
4. 统一任务反馈组件
5. 简历页交互闭环
6. 职位看板筛选与批量操作重构

## 10. 完成标准

当以下条件成立时，可以认为前端完成了第一轮“从可演示到可稳定使用”的改造：

- 主页面在 `375px` 下无横向滚动
- 页面不再依赖固定视口高度硬撑布局
- 主交互路径支持键盘完整操作
- 模态框具有正确语义和焦点管理
- 所有长任务都能在当前页面看到状态变化
- 职位看板具备筛选、搜索和更合理的批量操作路径
- 聊天、配置、简历三个页面的视觉和交互规则一致
