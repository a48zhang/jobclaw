# 模拟面试 SOP (Mock Interview)

### 场景
当用户要求“模拟面试”“根据这个岗位来面我”“扮演面试官”时，执行本 SOP。

### 步骤
1. **准备信息**：
   - 读取 `data/userinfo.md`。
   - 如果 `data/resume.typ` 存在，读取它。
   - 如果 `data/uploads/resume-upload.pdf` 存在，可使用 `read_pdf` 提取文本作为当前简历版本。
   - 若用户给出岗位 JD 或职位描述，按该岗位定制问题；否则按通用技术岗进行。
   - 若岗位信息会显著影响问题质量，使用 `request` 工具询问目标岗位或 JD；否则先通用面试再视情况追问。
2. **开始面试**：
   - 一次只问一个问题。
   - 问题顺序可以参考：
     - 自我介绍
     - 项目/实习经历讲解
     - 深挖简历内容
     - 计算机基础知识
     - 算法 / LeetCode
   - 每个问题都使用 `request` 工具获取用户回答，保持回合式节奏。
3. **过程控制**：
   - 回答过程中不要给完整点评和分数。
   - 每轮只允许简短过渡，例如“继续”“这个点我再深挖一下”。
   - 根据用户回答质量决定追问、跳题、降难度或加难度。
4. **结束时机**：
   - Agent 判断信息已经足够形成评价时，可以主动结束。
   - 用户明确说“结束面试”时，必须立即结束并进入总结。
   - 也可用 `request` 的 `confirm` 形式询问“是否结束面试”来收尾。
5. **结束总结**：
   - 统一输出：
     - 总分
     - 各子项得分
     - 全程表现分析
     - 推荐答案（仅针对答得不好的问题或模块）
     - 改进建议

### 注意事项
- 面试过程是回合式对话，不是一次性列一长串问题。
- 不要在中途生成最终结论。
- 推荐答案要贴近用户真实背景，不能替用户编造经历。
